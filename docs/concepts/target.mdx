---
title: "Target"
---

`Target` specifies the index or collection within our sink where we want to store our vectors. Retake provides
lightweight wrappers around targets for all supported sinks.

## ElasticSearch

**Usage**

```bash
target = Target.ElasticSearch(
    index_name="faqs",
    field_name="question_answer_embedding",
    should_index=True,
    similarity="cosine",
)
```

**Parameters**

<ParamField body="index_name" type="string" required>
  The name of the index where you want to store your embeddings
</ParamField>
<ParamField body="field_name" type="string" required>
  The field name of embedding
</ParamField>
<ParamField body="should_index" type="boolean">
  If the provided index and field name above do not exist, Retake will create an
  index with the provided field name. In this case, you must specify whether
  this field should be indexed. If the index and field name already exist, this
  parameter is ignored.
</ParamField>
<ParamField body="similarity" type="string">
  This field is only required if `should_index=True` and is used only if the
  provided index and field name above do not exist. Options for similarity are
  `l2_norm`, `dot_product`, and `cosine`, as per ElasticSearch documentation.
</ParamField>
