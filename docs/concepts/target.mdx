---
title: "Target"
---

`Target` specifies the index or collection within our sink where we want to store our vectors. Retake provides
lightweight wrappers around targets for all supported sinks.

## Pinecone

**Usage**

```bash
target = Target.Pinecone(
    index_name="faqs",
    namespace="question_answer_embedding"
)
```

**Parameters**

<ParamField body="index_name" type="string" required>
  Pinecone index name
</ParamField>
<ParamField body="namespace" type="string" required>
  Pinecone namespace
</ParamField>

## Weaviate

**Usage**

```bash
target = Target.Weaviate(
    index_name="example_class",
    default_vectorizer="text2vec-huggingface",
    default_vectorizer_config={
        "model": "sentence-transformers/all-MiniLM-L6-v2",
        "options": {"waitForModel": True, "useGPU": True, "useCache": True},
    }
)
```

**Parameters**

<ParamField body="index_name" type="string" required>
  Weaviate class name. `index_name` is used for the sake of naming consistency
  with other `Sink` types.
</ParamField>
<ParamField body="default_vectorizer" type="string" required>
  If the provided class name above does not exist, Retake will create a class
  with the provided class name. In this case, you must specify the vectorizer.
  Options for vectorizers are provided in the [Weaviate
  documentation](https://weaviate.io/developers/weaviate/modules/retriever-vectorizer-modules).
  If the class already exists, this parameter is ignored.
</ParamField>
<ParamField body="default_vectorizer_config" type="string" required>
  The vectorizer `moduleConfig` (example in the [Weaviate
  documentation](https://weaviate.io/developers/weaviate/modules/retriever-vectorizer-modules/text2vec-huggingface)).
  If the class already exists, this parameter is ignored.
</ParamField>

## ElasticSearch

**Usage**

```bash
target = Target.ElasticSearch(
    index_name="faqs",
    field_name="question_answer_embedding",
    should_index=True,
    similarity="cosine",
)
```

**Parameters**

<ParamField body="index_name" type="string" required>
  The name of the index where you want to store your embeddings
</ParamField>
<ParamField body="field_name" type="string" required>
  The field name of embedding
</ParamField>
<ParamField body="should_index" type="boolean">
  If the provided index and field name above do not exist, Retake will create an
  index with the provided field name. In this case, you must specify whether
  this field should be indexed. If the index and field name already exist, this
  parameter is ignored.
</ParamField>
<ParamField body="similarity" type="string">
  This field is only required if `should_index=True` and is used only if the
  provided index and field name above do not exist. Options for similarity are
  `l2_norm`, `dot_product`, and `cosine`, as per ElasticSearch documentation.
</ParamField>
