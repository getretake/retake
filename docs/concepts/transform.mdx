---
title: "Transform"
---

`Transform` specifies which table and columns contain our inputs and how we want to transform our inputs before
embedding them. Retake provides lightweight wrappers for transforms for all supported sources.

## Postgres

**Usage**

```bash
from typing import Union, Dict


def transform_func(question: Union[str, None], answer: Union[str, None]) -> str:
    return (question or "") + (answer or "")


def optional_metadata(
    question: Union[str, None], answer: Union[str, None]
) -> Dict[str, str]:
    return {"any_key": "any_value"}


transform = Transform.Postgres(
    # Note: Your table must have a primary key
    primary_key="id",
    relation="faqs",
    columns=["questions", "answers"],
    transform_func=transform_func,
    optional_metadata=optional_metadata,
)
```

**Parameters**

<ParamField body="primary_key" type="string" required>
  Postgres table primary key
</ParamField>
<ParamField body="relation" type="string" required>
  Postgres table name
</ParamField>
<ParamField body="columns" type="string" required>
  Columns that you want to listen on/transform
</ParamField>
<ParamField body="transform_func" type="function" required>
  Function that takes in columns as arguments and returns a single transformed
  string. This function is applied per-row.
</ParamField>
<ParamField body="optional_metadata" type="function">
  Optional function that takes in columns as arguments and returns a `Dict`
  specifying what metadata we want to add to our embedding. This function is
  applied per-row.
</ParamField>
